Detailed Description

template<typename BaseEnumT, typename ExtensionEnumT, BaseEnumT defValue = BaseEnumT(0)>
class ExtendedEnum< BaseEnumT, ExtensionEnumT, defValue >

This template can be used to extend an existing enum with new values.

Example, extend QNetworkReply::NetworkError with 2 new values: enum ICloudServiceError : int { CSE_GenericParsingError = QNetworkReply::NetworkError::UnknownServerError+1, CSE_FileCreationError, };

typedef ExtendedEnum<QNetworkReply::NetworkError, ICloudServiceError> CloudServiceError;

It can now be used as such: void func(QNetworkReply::NetworkError err) void func(CloudServiceError err) void func(ICloudServiceError err)

switch(err.value()) { case QNetworkReply::NoError: case CSE_GenericParsingError: }

if (err == QNetworkReply::NoError) if (err == CSE_GenericParsingError)

etc...

Note
There is no way for the compiler to ensure that the enum values do not overlap!
