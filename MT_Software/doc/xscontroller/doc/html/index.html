Xsens Device API Public Communication Objects
Xsens Device API low level communication library

Introduction

This library contains classes for communicating with Xsens devices on a basic level. It contains direct I/O interface classes as well as some message handling classes.

The target audience of this library is developers for small embedded platforms for which we do not have full XsensDeviceAPI support (yet). If you're using Windows or Linux on a PC platform, please use the full XDA dll/shared object instead.

These objects are the communication objects used by XDA, so the code can be read as a reference for interested people to see how XDA communicates with devices. If you find an error in the code, please let us know through suppo.nosp@m.rt@x.nosp@m.sens..nosp@m.com so we can improve the next version.

This library also contains the objects normally in the xstypes library. Since a user of this library does not have access to dlls or shared objects (otherwise he would be using XDA), this should not cause problems with multiple definitions.

Usage

The library consists of a folder structure with header and source files. All files that you need should be added to your project, as well as those that they depend on. The folder structure is important and should be maintained, otherwise you may need to update a lot of include statements. Also, add the base source file path of the library to your search path.

When building on Windows, make sure you have winusb available. We have included a version with the library.

In general, you will have to figure out platform-specific issues for yourself. If you encounter something that could be improved or a small tweak that we could make to allow easier migration to your specific platform, please let us know through suppo.nosp@m.rt@x.nosp@m.sens..nosp@m.com so we can improve the next version.
