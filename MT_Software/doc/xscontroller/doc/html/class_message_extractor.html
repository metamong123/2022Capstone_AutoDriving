Detailed Description

Helper class that extracts XsMessages from a stream of data. The user must call the processNewData function every time a new block of data is available. It is advised not to process too small blocks of data (e.g. per byte); Every single message must not span more than m_maxIncompleteRetryCount blocks to guarantee correct operation.

A MessageExtractor object maintains a buffer representing a sliding window over the data stream that is just big enough to contain any incompletely received XsMessage. The user can explicitly clear this buffer using the clearBuffer function
