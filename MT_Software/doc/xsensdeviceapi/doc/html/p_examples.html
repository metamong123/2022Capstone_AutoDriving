Code Examples
The SDK contains a set of example applications that show how to interface with the Xsens Device API and embedded examples that show how to control the MTi 1-series module from an embedded system, using the SPI or I2C bus.

These examples can be found in the Examples directory of the MT SDK:

<MT Software Suite installation directory>/MT SDK/Examples

XDA MTi Examples

The Xsens Device API MTi examples are divided into two types:
  • receive data example
  • parse log file example
They are both command line applications and both available in several programming languages: C++, C#, Matlab and Python. The structure of these examples is generally the same in all available programming languages.

Receive data example shows how to do following things:
  • Scan for MTi devices
  • Open the port for MTi
  • Configure MTi
  • Use MTi callbacks to receive live data
  • Read the incoming data from MTi
  • Record the data to a log file
Parse log file example shows how to do following things:
  • Open and load a log file
  • Use device callbacks for load progress update
  • Read the data from a log file
  • Export the data to a text file

XDA MTw Examples

The Xsens Device API MTw examples are divided into two types:
  • simple MTw example
  • Awinda monitor example
They are both available in several programming languages: C++, C# and Matlab.

Simple MTw examples are a command line applications that shows following things:
  • Scan and open a port for a wireless master (Awinda Station)
  • Connect MTw devices to a wireless master
  • Configure MTw's
  • Use wireless master callbacks to observe connectivy status of MTw's
  • Use MTw callbacks to receive live data
  • Read the incoming data from MTw's
Awinda monitor example is a complex end-user GUI application that shows how to implement a tool for managing Awinda stations and MTw devices, start/stop measures and record data. As a graphical library C++ example uses Qt and C# example uses WinForms.

Embedded Examples

The embedded examples show how to control the MTi 1-series module from an embedded system, using the SPI or I2C bus. The examples are written in C/C++ and can be built using either IAR or GCC.

There are currently two examples:
  • measurement data example (example_mti1_i2c_spi_receive_measurement_data)
  • protocol explorer (example_mti1_i2c_spi_mtssp_protocol_explorer)
The measurement data example is to show the typical program flow required to read measurement data from the device using the MTSSP protocol. This example is typically the first example to try out.

The protocol explorer is to gain some more insight in the MTSSP protocol, by interactively sending low level commands to the device and viewing the response from the device. With this example it is easier to analyze low level problems which may occur while embedding an MTi 1-series device in your solution.

More information about requirements and setup for these examples you can find in:

<MT Software Suite installation directory>/MT SDK/Examples/embedded_examples/readme.txt
