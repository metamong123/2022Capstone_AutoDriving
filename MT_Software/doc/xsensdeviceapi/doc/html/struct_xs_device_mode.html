Detailed Description

Contains the output configuration of a device.

Remarks
Skipfactor

The skip factor can be used to achieve lower update rates than the sample rate. For most update rates setUpdateRate will successfully come up with a period and skip factor that results in the correct sample rate with output skip factor. In some cases the resulting values may not be desirable. When this happens, use this function in combination with setPeriod or setUpdateRate.

The skip factor determines the update rate relative to the actual sample rate as

For example, to achieve an output rate of 25 Hz, one could write

XsDeviceMode mode;
mode.setUpdateRate(25);

or more explicitly:

XsDeviceMode mode;
mode.setUpdateRate(100);
mode.setSkipFactor(3);
Note
MT Mk4 devices are better configured using XsOutputConfiguration, this object is supplied for legacy support only.
