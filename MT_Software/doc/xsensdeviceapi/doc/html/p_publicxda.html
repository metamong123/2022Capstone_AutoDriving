Public Xsens Device API

Introduction

The public Xsens Device API (XDA) is an open-source library to easily communicate with MTi devices on various platforms. This library is made for our customers who use Xsens industrial motion trackers (MTi's) with embedded Linux/Windows. In this document, we will give you a brief overview of public XDA.

Overview

Application development set-up

The public XDA comes from the original XDA, containing only the parts concerning communication with MTi devices and recording/replaying MTB files. If you want a more in-depth look at the original XDA - checkout the Overview of the Xsens Device API document.

This public XDA library contains following functionality:

  • Device discovery - various scanners to scan for MTi devices on different interfaces, such as RS232 and USB.
  • Device classes - providing a C/C++ interface for each MTi device.
  • Serial communication - handles all low-level communication with devices.
  • Packet parsing - parsers which parse raw data from a device or file into distinct messages, for example for the Xbus protocol.
  • MTB files handling - performs all I/O operations to record and/or load MTB files.
  • Journaller - a logger for debugging.

The source code consists of the following packages:

  • xscontroller: containing device classes, communicators, scanners and data parsers.
  • xscommon: : containing utilities for threading and logging.
  • xstypes: defining generic data types (vectors, matrices, quaternions, etc.).

Requirements

In theory, you should be able to use public XDA on any OS that has a C/C++ compiler and a USB port. Nevertheless, if you want to be sure that public XDA will be built and run properly, your system should meet the following tested requirements:

  • Operating system: 32-bit/64-bit/ARM Linux or Windows
  • C/C++ Compiler: GCC 5.4.0 or MSVC 14.0
  • C++ Version: C++11

Getting started

Make sure that you have Xsens MT SDK installed on your system. Before you start using the public XDA, we recommend taking a look at the 'Public XDA Examples', available in the Examples folder from installed MTSDK directory. So you will have some idea how to use it in your application.

All information about how to compile and link a program can be found in either the Visual Studio Solution file or the Makefiles, located in 'src_cpp' example folder. Or you can simply copy xspublic folder, which contains Makefiles, from MTDSK directory to your application directory and start developing.
