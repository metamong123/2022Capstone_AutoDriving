Upgrading from XDA 4.x

Upgrading from XDA 4.0 to XDA 4.1 or higher

We always try to avoid making breaking changes in our XDA interface. However, sometimes it's necessary to make some changes to preserve quality. This section gives a quick overview of potentially breaking changes and some new features in the latest XDA.

Breaking changes

  • The "hardware" scenario functions have been renamed to "onboard" scenario functions.
  • The onMeasurementStarted, onMeasurementStopping, onMeasurementStoppped, onWaitingForRecordingStart, onRecordingStarted and onRecordingStopped callbacks have been replaced by XsCallback::onDeviceStateChanged
  • XsIoHandle is now in file xsfilepos.h together with XsFileHandle
  • The internal storeAlignmentMatrix function has been removed from the XsDevice interface.
  • The XsQueueMode is now a mutually exclusive choice of XQM_CallbackOnly, XQM_Fifo or XQM_Last.
  • The coordinateSystem function of the XsDataPacket interface is replaced by the coordinateSystemOrientation function.

New features

  • Support for the MTi-G-700 device.
  • The XsDataBundler class is now available for receiving data from several devices at the same time. Please note that devices that are not synchronized through a hardware connection will have time drift with respect to each other. MTws connected to the same Awinda Station are synchronized through wireless hardware.
  • The XsCallback::onConnectivityChanged, XsCallback::onInfoResponse and XsCallback::onError callbacks have been added.
  • It is now possible to configure synchronization features and XsDevice::supportedSyncSettings has been added to provide available synchronization options per device. Please be sure to use at least firmware version 1.1 for MT Mk4 devices or synchronization will not be available for your device.
  • It is now possible to set and request XsQueueMode globally and per XsDevice.
  • For basic Awinda support, several functions and communication structures have been added to XsDevice.

Upgrading from XDA 4.4 to XDA 4.5 or higher

This section gives a quick overview of potentially breaking changes and some new features in the latest XDA.

Breaking changes

  • XsControl::setSerialKey has been removed
  • XsDataBundler has been removed. This is now standard functionality of the container devices Xbus Master, Bodypack and the various Awinda Station variants.
  • callback onDataAvailable has been renamed to XsCallback::onLiveDataAvailable
  • callback onWriteDataToLogFile has been removed. It has been partially replaced by XsCallback::onRecordingDataAvailable
  • XsQueueMode has been removed and all functions using it as well. It has been replaced with new XsOption items
  • Function xdaDllInfo is now in xdadll.h instead of in xdainfo.h
  • Function xdaDllVersion has been renamed to xdaVersion
  • The parameter for XsControl::flushInputBuffers has been removed
  • XsControl::libraryPath has been removed
  • XsControl::setOptionEnabled and XsDevice::setOptionEnabled functions have been replaced with XsControl::setOptions and XsDevice::setOptions, which allow setting and clearing multiple options at the same time
  • XsControl::isOptionEnabled and XsDevice::isOptionEnabled functions have been replaced with XsDevice::areOptionsEnabled, which allow checking for several options at the same time
  • When loading a file in XDA one has 2 options:
    • Retrieve data during loading of the file using callbacks
    • Retrieve data afterwards using the XsDevice::getDataPacketByIndex function for the XsDevice::getDataPacketCount number of times instead of XsDevice::queueLength.
    For the second option the XSO_RetainBufferedData has to be enabled. This keeps the recording data in a cache so it can be accessed through XsDevice::getDataPacketByIndex
  • XsDevice::isMasterDevice now returns true if the device is a master device. XsDevice::isContainerDevice has been added to detect if a device can contain child devices
  • The functionality of XsDevice::queueLength has been slightly changed so it will include the number of packets that still need to arrive, not just the number of packets in the queue. It is been renamed to XsDevice::recordingQueueLength.
  • XsDevice::cacheSize returns the plain number of items in the cache.
  • XsOption enum values have been updated to be bitfield compatible. XSO_None has been added and XSO_Count has been removed

New features

  • Callbacks onAllLiveDataAvailable and onAllRecordingDataAvailable have been added to replace removed XsDataBundler functionality.
  • Support Awinda System 2nd generation. (Starting XDA 4.5.3)

Upgrading from XDA 4.6 to XDA 4.7 or higher

Breaking changes

  • 'recording data' was renamed to 'buffered data' and 'recorded data' and a general 'data' callback were defined. This is the biggest breaking change in this version, which was required to make the callback behaviour predictable and consistent. The general 'data' callbacks are advised unless you want to do advanced multi-stream parallel data visualization/handling.
  • XsCallback::onRecordingDataAvailable has been renamed to onBufferedDataAvailable (also applies to AllRecordingAvailable)
  • A new item was added to XsConnectivityState: XCS_WirelessOutOfRange that changed the value of XCS_File and XCS_Unknown
  • XsOption received some new options that changed the value of XSO_All and we renamed XSO_RetainRecordingData to XSO_RetainBufferedData and XSO_OrientationInRecordingStream to XSO_OrientationInBufferedStream
  • XsStatusFlag::XSF_Retransmitted's value changed and a new XSF_ClippingDetected was added that uses the old value of XSF_Retransmitted
  • Some of the XsComCallbackOptions were renamed in line with the change from 'recording data' to 'buffered data'
  • IXsensControlApi::live/recordingDataPacket(s)Handled were replaced by the more generic dataPacket(s)Handled

New features

New features that were added without breaking existing functionality:

  • XsCallback::on(All)DataAvailable
  • XsCallback::on(All)RecordedDataAvailable
  • XsControl::libraryPath, XsControl::enabledOptions, XsControl::disabledOptions, XsControl::setOptionsForce
  • XsDevice::isStandaloneDevice, XsDevice::getStopRecordingPacketId, XsDevice::getOptions, XsDevice::operationalMode, XsDevice::setOperationalMode, XsDevice::deviceBufferSize, XsDevice::setDeviceBufferSize, XsDevice::waitForAllDevicesInitialized, XsDevice::startRepresentativeMotion, XsDevice::representativeMotionState, XsDevice::stopRepresentativeMotion, XsDevice::storeIccResults, XsDevice::isLoadLogFileInProgress, XsDevice::waitForLoadLogFileDone
  • XsDeviceOptionFlag has some new items
  • XsDeviceRef has some more comparison operators
  • ICC and Representative Motion has been added
  • Two new string types were added to XsNmeaStringType: XNST_XSVEL and XNST_GPZDA
